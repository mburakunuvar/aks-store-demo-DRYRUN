## Exercise 2: Container Image Scan Workflow (Medium)

Create a Trivy-based container scan workflow. Available pre-built in `checkpoint/complete`.

**Option A — Quick**: Merge `checkpoint/complete` into `main` to get the file automatically.

**Option B — Manual**: Create `.github/workflows/docker-scan.yml` via github.com UI with this content:

```yaml
name: "Container Image Scan → Auto-Create Issues → Assign to Copilot"

on:
  schedule:
    - cron: "0 9 * * 1" # Weekly on Monday at 9am UTC
  workflow_dispatch:

permissions:
  contents: read
  issues: write

jobs:
  scan:
    name: Scan ${{ matrix.service }}
    runs-on: ubuntu-latest
    strategy:
      fail-fast: false
      matrix:
        service:
          - order-service
          - makeline-service
          - product-service
          - ai-service
    steps:
      - uses: actions/checkout@v4

      - name: Build image
        run: docker build -t ${{ matrix.service }}:scan src/${{ matrix.service }}

      - name: Run Trivy vulnerability scanner
        uses: aquasecurity/trivy-action@0.28.0
        with:
          image-ref: "${{ matrix.service }}:scan"
          format: "json"
          output: "trivy-results.json"
          severity: "CRITICAL,HIGH"

      - name: Parse results and create issue
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const service = '${{ matrix.service }}';

            let results;
            try {
              results = JSON.parse(fs.readFileSync('trivy-results.json', 'utf8'));
            } catch (e) {
              console.log('No Trivy results file or invalid JSON, skipping');
              return;
            }

            const vulns = (results.Results || [])
              .flatMap(r => r.Vulnerabilities || [])
              .filter(v => v.Severity === 'CRITICAL' || v.Severity === 'HIGH');

            if (vulns.length === 0) {
              console.log(`No CRITICAL/HIGH vulnerabilities found in ${service}`);
              return;
            }

            const existing = await github.rest.issues.listForRepo({
              owner: context.repo.owner,
              repo: context.repo.repo,
              labels: `security,automated,containers,${service}`,
              state: 'open'
            });
            if (existing.data.length > 0) {
              console.log('Open issue already exists, skipping creation');
              return;
            }

            const findings = vulns.slice(0, 15).map(v =>
              `- **${v.PkgName}** ${v.InstalledVersion} → ${v.FixedVersion || 'no fix yet'} (${v.Severity}): ${v.Title || v.VulnerabilityID}`
            ).join('\n');

            const total = vulns.length;
            const truncated = total > 15 ? `\n\n_...and ${total - 15} more. Run Trivy locally for full results._` : '';

            await github.rest.issues.create({
              owner: context.repo.owner,
              repo: context.repo.repo,
              title: `[Security] ${total} container vulnerabilities in ${service}`,
              body: [
                '## Container Image Scan Finding',
                '',
                `**Image:** ${service}`,
                '**Scanner:** Trivy',
                `**Found:** ${total} CRITICAL/HIGH vulnerabilities`,
                '',
                '### Top Findings',
                findings,
                truncated,
                '',
                '### Remediation',
                `- Update base image in \`src/${service}/Dockerfile\` to a newer version`,
                '- Update vulnerable OS packages or application dependencies',
                '- Rebuild and re-scan to verify fixes',
                '',
                '> This issue was auto-generated by the container scan workflow.',
              ].join('\n'),
              labels: ['security', 'automated', 'containers', service],
              assignees: ['copilot']
            });
```

After committing, go to **Actions** → **"Container Image Scan"** → **"Run workflow"** to trigger it.

## Exercise 4: Branch Protection Rules (Easy)

Set up branch protection to enforce the agentic SDLC pipeline:

1. Go to your fork → **Settings** → **Branches** → **Add branch protection rule**
2. Branch name pattern: `main`
3. Enable:
   - ✅ Require a pull request before merging
   - ✅ Require approvals (1)
   - ✅ Require status checks to pass before merging (select your CI workflows)
   - ✅ Require Copilot code review (if available)
4. Save

Now every PR — whether from Copilot or a human — must pass CI and get reviewed before merge.

## Exercise 5: Extend the Pipeline (Advanced)

Add more scanners to the `security-audit-autofix.yml` workflow:

**Go vulnerability scanning** — add a new job:

```yaml
audit-go:
  name: Audit Go Dependencies
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-go@v5
      with:
        go-version: "1.24"
    - name: Run govulncheck
      id: audit
      run: |
        go install golang.org/x/vuln/cmd/govulncheck@latest
        cd src/makeline-service
        set +e
        govulncheck ./... > vulncheck-output.txt 2>&1
        VULN_EXIT=$?
        set -e
        echo "exit_code=$VULN_EXIT" >> "$GITHUB_OUTPUT"
    - name: Create issue if vulnerabilities found
      if: steps.audit.outputs.exit_code != '0'
      uses: actions/github-script@v7
      with:
        script: |
          const fs = require('fs');
          const output = fs.readFileSync('src/makeline-service/vulncheck-output.txt', 'utf8').trim();
          const existing = await github.rest.issues.listForRepo({
            owner: context.repo.owner,
            repo: context.repo.repo,
            labels: 'security,automated,makeline-service',
            state: 'open'
          });
          if (existing.data.length > 0) return;
          await github.rest.issues.create({
            owner: context.repo.owner,
            repo: context.repo.repo,
            title: '[Security] Go vulnerability findings in makeline-service',
            body: `## Automated Go Vulnerability Scan\n\n**Service:** makeline-service\n**Scanner:** govulncheck\n\n### Findings\n\`\`\`\n${output.substring(0, 3000)}\n\`\`\`\n\n> Auto-generated by security audit workflow.`,
            labels: ['security', 'automated', 'makeline-service'],
            assignees: ['copilot']
          });
```

**Rust vulnerability scanning** — add another job:

```yaml
audit-rust:
  name: Audit Rust Dependencies
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - name: Install cargo-audit
      run: cargo install cargo-audit
    - name: Audit product-service
      id: audit
      run: |
        cd src/product-service
        set +e
        cargo audit --json > audit-output.json 2>/dev/null
        AUDIT_EXIT=$?
        set -e
        echo "exit_code=$AUDIT_EXIT" >> "$GITHUB_OUTPUT"
    - name: Create issue if vulnerabilities found
      if: steps.audit.outputs.exit_code != '0'
      uses: actions/github-script@v7
      with:
        script: |
          const fs = require('fs');
          const existing = await github.rest.issues.listForRepo({
            owner: context.repo.owner,
            repo: context.repo.repo,
            labels: 'security,automated,product-service',
            state: 'open'
          });
          if (existing.data.length > 0) return;
          await github.rest.issues.create({
            owner: context.repo.owner,
            repo: context.repo.repo,
            title: '[Security] Rust dependency vulnerabilities in product-service',
            body: '## Automated Rust Vulnerability Scan\n\n**Service:** product-service (`src/product-service/`)\n**Scanner:** cargo-audit\n\nUpdate vulnerable dependencies in `Cargo.toml` and run `cargo update`.\n\n> Auto-generated by security audit workflow.',
            labels: ['security', 'automated', 'product-service'],
            assignees: ['copilot']
          });
```
